M5C3SE_MQTT 使用手冊

本系統專為 M5Stack CoreS3 設計，提供強大的 MQTT 數據監控、雙網路切換及手動校時功能。

1. 啟動與自動運行 (Auto-Run)

開機後進入引導界面。若 30 秒 內無觸碰操作，系統將自動啟動：

  • 連線記憶：系統會自動使用最後一次成功連線的介面（WIFI 或 LAN）。
  • 連線診斷：在 Connecting... 畫面會以黃色字體顯示當前的 SSID、密碼及連線模式，便於排查 SD 卡設定問題。
  • 防衝突機制：若已手動進入 UPDATE 模式，自動運行計時器將自動禁用，確保更新不被中斷。

2. MQTT 監控模式 (RUN)

  • 點擊 RUN 進入網路選擇界面。
  • WIFI：掃描並連接無線網路（支持多頁滾動選擇）。
  • LAN (DHCP)：使用有線模組自動獲取 IP。
  • LAN (STATIC)：手動輸入靜態 IP 配置。

3. 歷史紀錄回顧 (Review)

進入監控界面後，系統會自動保留最近 10 筆 接收到的消息：

  • 點擊右下角 UP：查看上一筆歷史數據（修正回溯邏輯）。
  • 點擊右下角 DN：查看較新的數據。翻至最新一筆時會自動切換回實時監控。
  • 畫面頂部會顯示 [1] ~ [10] 提示當前回顧索引。

4. 設定功能

A. 系統校時 (SET RTC)

點擊 SET RTC。使用 + / - 調整數值，Next 切換欄位，最後按 SAVE 寫入硬件。

B. MQTT 參數設定 (SET MQTT)

點擊 SET MQTT 進入引導式設定：

  1. 輸入 Server 地址 -> 按 OK。
  2. 輸入 Port (預設 1883) -> 按 OK。
  3. 輸入訂閱 Topic (SUB) -> 按 OK。
  4. 輸入發布 Topic (PUB) -> 按 OK 自動儲存至 SD 卡。

💡 鍵盤說明：KB 切換英/數分頁；HOME 隨時放棄設定回到首頁。

5. 固件更新 (UPDATE)

點擊 UPDATE。系統將強制開啟 WiFi 並連線至 GitHub 檢查最新固件。

注意：自 v1.1.12 起，進入此模式後會自動鎖定背景計時器，避免與 30 秒自動運行衝突導致更新失敗。
