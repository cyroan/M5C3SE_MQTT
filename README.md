# M5C3SE_MQTT 开发纪录

本项目是基于 **M5Stack CoreS3** 开发的 MQTT 监控与 OTA 更新系统，支持双网切换及离线历史回顾。

---

## 开发规范 (Development Guidelines)

### 1. 版本号管理 (Versioning)
- **文件位置**：`version.h` (`#define FIRMWARE_VERSION`)
- **变更原则**：每次代码修改或 Bug 修复后，必须将 **Minor Version 第三位** 递增（例如 `1.1.9` -> `1.1.10`）。
- **同步要求**：需同步更新 `version.txt` 以确保 OTA 系统能识别最新版本。

### 2. 开发环境与工具链
- **编译器**：使用 `arduino-cli` 进行本地编译。
  - 核心 FQBN: `esp32:esp32:m5stack_cores3`
- **版本控制**：使用 `git` 进行管理，代码及编译后的 `.bin` 文件需同步推送至 GitHub 仓库。
- **本地烧录**：通过 USB (如 `COM10`) 使用 `arduino-cli upload` 进行快速测试。
- **远程更新**：通过 GitHub HTTPS 实现无线 OTA 更新。

## 版本演进 (Development Log)

### v1.1.15 (Latest)
- **UI 交互增强**：新增專用文字捲動按鈕（`^` 與 `v`），單次點擊可跳轉約 5 行文字（80 像素），解決長訊息滑動不易的問題。
- **佈局優化**：調整訊息顯示區域高度，避免與底部導覽列重疊，並優化捲軸視覺效果。

### v1.1.14
- **長訊息捲動支援**：引入 `M5Canvas` 與 `ClipRect` 技術，實現 MQTT 訊息區域的垂直捲動功能。
- **狀態管理**：切換歷史紀錄時自動重置捲動位置。

### v1.1.13
- **MQTT 性能修復**：將 MQTT 接收緩衝區 (`setBufferSize`) 擴大至 **2048 bytes**，解決 1KB 以上長訊息收不到的問題。
- **深度偵錯支援**：加入 Serial Debug 日誌，包含連線狀態、訂閱結果、以及接收訊息的 Topic 與長度資訊。

### v1.1.12
- **流程加固**：彻底解决 OTA 更新与 30 秒自动运行计时器冲突的问题。在 `UPDATE` 模式下自动禁用超时跳转。
- **稳定性优化**：在 `loop()` 增加状态锁，防止在关键任务（如烧录中）发生非预期的状态切换。

### v1.1.11
- **连线诊断 UI**：在 `Connecting...` 界面增加连线参数实时显示（SSID、Password、Network Mode）。
- **调试支持**：方便开发者快速确认从 SD 卡读取的配置信息是否正确。

### v1.1.10
- **自动运行完全修复**：解决了自动进入时 WiFi 硬件未初始化 (`WIFI_STA`) 的问题。
- **网络偏好持久化**：新增 `/net_pref.txt`，记忆最后一次成功连线的介面（WiFi/LAN），确保启动后自动选择正确的物理链路。
- **启动延时**：在 `setup()` 增加 SD 卡初始化前的延时，提高冷启动时的读取成功率。

### v1.1.9
- **逻辑重构**：修正 30 秒超时后直接跳过连接步骤的 Bug，现在会执行正规的 `STATE_CONNECTING` 流程。
- **网络安全检查**：在 MQTT 重连前强制检查网络物理连通性，消除因空指针或未联网导致的死机。
- **历史记录修复**：修正监控界面 `UP` 按钮的历史记录回溯逻辑。

### v1.1.8
- **状态转换修复**：修正 `loop()` 中逻辑导致的启动页面反复跳变 Bug。
- **触控优化**：缩小 `STATE_RUNNING` 下 HOME 键的判定面积，减少误触。

### v1.1.0 - v1.1.7 (核心摘要)
- **键盘 UI**：功能键字体加大至 Size 2，导航逻辑重更名为 `KB` 与 `HOME`。
- **MQTT 引导化**：Server -> Port -> Sub -> Pub 分步设置。
- **双网支持**：兼容 WiFi 及 LAN Module 13.2。
